# FounderLab AI — Master Engineering Guide
Version: 1.0

This document defines the permanent engineering principles for FounderLab AI.

Every task, every commit and every line of code must follow these rules.

------------------------------------------------------------
MISSION
------------------------------------------------------------

FounderLab AI is NOT another AI chatbot.

FounderLab is the operating system for founders.

Every feature must save users time.

Every interaction must feel intelligent.

Every release must noticeably improve the product.

Never build average software.

------------------------------------------------------------
PRIMARY OBJECTIVE
------------------------------------------------------------

Build the highest quality AI workspace possible.

Quality is always more important than speed.

Never sacrifice architecture for temporary fixes.

Avoid technical debt whenever possible.

------------------------------------------------------------
ENGINEERING PRINCIPLES
------------------------------------------------------------

Always prefer

Clean Architecture

Simple code

Reusable components

Maintainability

Scalability

Performance

Security

Accessibility

Every feature should feel production ready.

------------------------------------------------------------
RULE ZERO
------------------------------------------------------------

Never begin another feature until the current feature is fully working.

Broken software is never acceptable.

Fix.

Test.

Verify.

Only then continue.

------------------------------------------------------------
WORKFLOW
------------------------------------------------------------

For every task:

1. Understand the request completely.

2. Inspect the existing codebase.

3. Find the root cause.

4. Design the cleanest solution.

5. Implement.

6. Test.

7. Verify.

8. Refactor if needed.

9. Only then finish.

Never skip steps.

------------------------------------------------------------
NO PLACEHOLDERS
------------------------------------------------------------

Never leave

TODO

FIXME

Temporary UI

Mock buttons

Fake loading

Fake AI

Fake success messages

Every visible feature should actually work.

------------------------------------------------------------
FOUNDERLAB MODULES
------------------------------------------------------------

FounderLab contains intelligent modules.

Every module must evolve continuously.

Never consider any module "finished."

FINAL RULE
------------------------------------------------------------

Before every commit ask:

Would this impress a startup founder?

Build software people remember.

## FounderLab Philosophy

FounderLab exists to eliminate repetitive work.

Every feature should remove work from the user.

Never ask the user to perform manual steps if FounderLab can automate them safely.

Think before implementing.

Ask:

Can this be automated?

Can this be remembered?

Can this become one click?

Can AI remove this friction?

Always choose the solution that saves the user the most time while keeping the system reliable.

## Every Commit Must Improve FounderLab

Never make a commit that only completes the requested task.

While working, inspect nearby code.

If obvious improvements exist that are safe:

- improve naming
- remove duplication
- improve UX
- improve performance
- improve accessibility
- improve documentation

Leave the codebase cleaner than you found it.

This is the FounderLab Standard.
